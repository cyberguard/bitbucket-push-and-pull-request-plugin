Allow you to override the URL in the bitbucket payload from the webhook so that it match the url you define in the SCM configuration.

If you are seeing the message

    skipping resolution of commit eef18d94302f5a3945796e77ee8ec46a69fd1ddf, since it originates from another repository
then most likely the JSON posted by Bitbucket to your Jenkins Webhook has a URL of the type
    https://bitbucket.org/XXXX/YYYY
but your SCM Repository URL config has another url

This allows you to update the URL inside the JSON received from BitBucket to match that of your SCM config.

For example: 

        {
        "https://bitbucket.org/XXXX/YYYY" : "https://ZZZZ@bitbucket.org/XXXX/YYYY.git"
        }
would change the URL from https://bitbucket.org/XXXX/YYYY to https://ZZZZ@bitbucket.org/XXXX/YYYY.git before trying to match it to your SCM config.
